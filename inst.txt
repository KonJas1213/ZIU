🎯 Cel projektu
Zaprojektowanie i zaimplementowanie interfejsu użytkownika dla wybranej aplikacji — od prototypu lo-fi, przez projekt hi-fi, aż po działające wdrożenie.

Projekt ma pokazać, że student potrafi świadomie podejmować decyzje projektowe (UX/UI), implementować je zgodnie z aktualnymi standardami webowymi lub mobilnymi oraz oddać działający produkt — możliwy do uruchomienia przez każdego.

Domyślny temat to aplikacja To-Do / Task Manager rozwijana wspólnie na pierwszych laboratoriach. Student może jednak zaproponować własny temat — aplikację webową, mobilną lub hybrydową — pod warunkiem zgłoszenia go do Lab 9 (do 08.05.26, przez Teams lub bezpośrednio u prowadzącego).

📋 Zakres projektu i punktacja
Maksymalna liczba punktów: 50 pkt

1. Prototypowanie UI — 6 pkt
Prototyp lo-fi (szkice, wireframes lub narzędzie do mockupów) (0–2)
Prototyp hi-fi z przepływem ekranów (user flow) (0–2)
Spójność projektu graficznego z finalną implementacją (0–2)
2. Implementacja interfejsu — 7 pkt
Podział aplikacji na komponenty wielokrotnego użytku (0–3)
Routing między widokami (min. 2–3 ekrany) (0–2)
Zastosowanie biblioteki UI (MUI lub uzgodnionej alternatywy) (0–2)
3. Responsive Design — 5 pkt
Poprawne działanie na urządzeniach mobilnych i desktopowych (0–2)
Min. 2 breakpointy lub natywne wzorce nawigacji (aplikacje mobilne) (0–1)
Ogólna jakość i spójność layoutu (0–2)
4. Formularze i walidacja — 5 pkt
Co najmniej jeden formularz z walidacją po stronie klienta (0–2)
Czytelne komunikaty błędów widoczne dla użytkownika (0–2)
Obsługa stanu formularza (React Hook Form / Zod lub odpowiednik) (0–1)
5. Dostępność — WCAG — 8 pkt
Semantyczny HTML, aria-labels, odpowiednie role (0–2)
Kontrast kolorów minimum AA (4,5:1) (0–2)
Nawigacja klawiaturą i widoczny fokus (0–2)
Pozytywny wynik audytu Lighthouse lub AXE (brak krytycznych błędów) (0–2)
6. State Management — 4 pkt
Globalny stan aplikacji (Context API, Redux, Zustand, Pinia lub odpowiednik) (0–2)
Obsługa stanów: loading · success · error (0–2)
7. Integracja z API lub mockiem — 5 pkt
Pobieranie i wysyłanie danych (GET + co najmniej jeden z: POST, PUT, DELETE) (0–3)
Alternatywnie: mock przez MSW, JSON Server, Requestly lub inne
Widoczna obsługa błędów sieciowych w UI (0–2)
8. Mikrointerakcje i animacje — 5 pkt
Animacje przejść między widokami (0–2)
Wizualny feedback akcji użytkownika (loading spinner, komunikat sukcesu/błędu) (0–2)
Zastosowanie Framer Motion, CSS transitions lub odpowiednika (0–1)
9. Deployment i dokumentacja — 5 pkt
Aplikacja wdrożona publicznie (Netlify, Vercel, GitHub Pages lub inne) (0–1)
Repozytorium GitHub z historią commitów (0–2)
Plik README.md zawierający: opis projektu, link do demo, instrukcję uruchomienia i użyte technologie (0–2)
Kryteria premiowane (nieobowiązkowe, +0.5 stopnia)
Notatka UX — krótki dokument (1–2 strony, może być częścią README.md) zawierający opis grupy docelowej lub personę użytkownika, uzasadnienie kluczowych wyborów UI/UX oraz odniesienie do heurystyk Nielsena, zasad UCD lub wyników własnych obserwacji.

Aktywność na zajęciach — terminowe oddawanie etapów pośrednich i aktywny udział w laboratoriach.
